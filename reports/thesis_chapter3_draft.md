# 第三章 基于神经符号架构的肺结节随访建议生成方法

## 3.1 问题定义与总体技术路线

在临床诊疗流程中，放射科医生出具的胸部 CT 报告是肺结节筛查与随访的核心依据。然而，这些报告通常以非结构化文本形式存储在放射学信息系统（RIS）中，表述风格因医生习惯而异，且包含大量复杂的医学术语与句法结构。如何从这些海量的非结构化报告中自动、准确地提取结节特征，并生成符合临床指南（如 Lung-RADS v2022 或 Fleischner Society 指南）的标准化随访建议，是实现智能化肺癌早筛管理的关键挑战。

目前，直接利用端到端（End-to-End）的生成式大语言模型处理此类任务面临多重困境。首先，临床随访指南具有极高的严谨性，任何细微的特征提取偏差都可能导致随访频率或处置建议的错误，而黑盒化的深度学习模型难以避免“幻觉”风险。例如，模型可能在没有明确证据的情况下，将一个微小的实性结节误判为需要立即活检的高危结节，这种不可控性在临床应用中是不可接受的。其次，医疗决策要求极高的可追溯性，纯生成式模型往往无法提供清晰的逻辑推理路径，难以满足临床审计的要求。当医生质疑系统的建议时，端到端模型无法给出具体的指南条款或特征依据。最后，真实世界的医疗数据（如 MIMIC-IV 数据集）存在严重的脱敏现象，尤其是吸烟史等关键高危属性的缺失，使得端到端模型在缺乏完整上下文时容易产生不可控的推断，甚至编造虚假的病史信息。

此外，从自然语言处理的角度来看，肺结节随访建议生成并非简单的文本摘要或翻译任务。它本质上是一个从高度非结构化的观察描述到高度结构化的决策逻辑的映射过程。端到端模型试图在单一的概率空间内同时完成语义理解与逻辑推理，这往往会导致两者在复杂场景下的相互干扰。

基于上述挑战，本章提出了一种**“神经-符号解耦架构”（Neuro-Symbolic Decoupled Architecture）**。该架构的核心思想是将复杂的随访建议生成任务拆解为两个相对独立的阶段：
1. **神经感知层（Neural Perception Layer）**：利用预训练语言模型（如 PubMedBERT）强大的语义理解能力，从非结构化文本中抽取结构化的临床事实（Radiology Facts）。这一层解决了医学文本表述多样性与模糊性的问题。
2. **符号决策层（Symbolic Decision Layer）**：将抽取出的结构化事实输入到基于专家规则构建的确定性推理引擎中。该引擎严格遵循临床指南逻辑，通过显式的决策树生成随访建议。

这种解耦设计在学术上具有显著优势。它借鉴了神经符号系统（Neuro-Symbolic Systems）的一般性原则，即结合了联结主义（Connectionism）在模式识别上的鲁棒性与符号主义（Symbolism）在逻辑推理上的严密性。通过这种方式，系统不仅能够处理复杂的临床语言变体，还能确保最终的医学决策始终锚定在权威指南的框架内。这种架构不仅提高了系统的可靠性，还为临床医生提供了一个可解释、可验证的辅助决策工具。

## 3.2 系统总体架构

本研究构建的肺结节随访建议生成系统由数据预处理、神经感知抽取、结构化信息聚合以及符号规则推理四个核心层次组成。系统总体架构如图 3-1 所示。

系统的具体工作流程如下：
1. **数据预处理层**：这是整个流水线的起点。首先，系统对原始的放射学报告进行过滤，通过精确匹配 `exam_name` 字段（如 "CT CHEST W/O CONTRAST" 或 "CT LUNG NODULE PROTOCOL"）筛选出胸部 CT 检查记录。随后，利用一套精心构建的结节相关关键词库（包含 "nodule", "mass", "opacity", "ground-glass" 等）对报告进行初步筛选，剔除无结节描述的记录。最后，通过段落解析器（Section Parser）识别并提取报告中的“FINDINGS”与“IMPRESSION”等核心章节。这一步骤至关重要，因为它能有效过滤掉报告中的行政信息、既往史回顾等干扰内容，为后续抽取提供聚焦的上下文。
2. **神经感知层**：这是系统的核心信息来源。其中，`radiology` 信息抽取模块负责从报告文本中提取结节的密度（`density_category`）、尺寸（`size_mm`）及位置（`location_lobe`）等关键形态学特征。该层采用了提及级（Mention-level）的抽取策略，能够处理同一报告中出现的多个结节。同时，吸烟史与高危筛查资格模块作为辅助组件，尝试从出院小结中推断患者的吸烟状态。
3. **结构化聚合层**：该层引入了 `case_bundle_assembler` 模块，其作用是将来自不同数据源、不同维度的抽取结果，按照患者 ID（`subject_id`）进行关联与聚合。信息流在此处汇合：放射学事实从单份报告级别提升至患者级别，并与吸烟史等背景信息对齐，最终构建出统一的中间表示——`case_bundle`。
4. **符号决策层**：这是系统的最终输出阶段。基于 Lung-RADS v2022 标准构建的规则引擎接收 `case_bundle` 作为输入。引擎内部维护了一套复杂的决策树逻辑，通过对聚合后的特征进行逐一判定，生成包含建议等级、指南依据及推理路径的标准化报告。
5. **工程补偿组件**：在感知层与决策层之间，系统嵌入了共享尺寸解析器（Shared Size Parser）。该组件作为一种工程补偿策略，专门负责将模型提取的文本边界转化为标准化的毫米数值。这种设计确保了后续规则引擎接收到的尺寸数据是精确且格式统一的，从而规避了模型在数值计算上的不稳定性。

通过这种层级化的设计，信息从原始的非结构化文本出发，经过清洗、抽取、聚合和推理，最终转化为具有临床指导意义的结构化建议。每一层都承担了明确的职责，且层与层之间通过标准化的 Schema 进行衔接，保证了整个系统的高效运行与易维护性。

## 3.3 结构化中间表示设计

为了实现神经感知与符号推理的有效衔接，本项目设计了一套严密的结构化中间表示协议，其核心载体为 `case_bundle`。这种设计不仅定义了模块间的接口契约，更在逻辑上实现了状态隔离。

### 3.3.1 数据主链路与并行补充链

系统的数据流转遵循两条清晰的路径，最终在 `case_bundle` 中完成汇合：
1. **数据主链路（Primary Data Path）**：`Radiology Report → radiology_fact → case_bundle → recommendation`。这是系统生成随访建议的核心路径。放射学报告经过感知层处理后，转化为单份报告级别的 `radiology_fact`。随后，这些事实被聚合到 `case_bundle` 中，作为规则引擎生成最终 `recommendation` 的主要依据。
2. **并行补充链（Parallel Supplementary Path）**：`Discharge Note → smoking_eligibility → case_bundle`。吸烟史与高危属性的提取作为一条并行的补充链条，从出院小结中获取信息并转化为 `smoking_eligibility`。这一链条为 `case_bundle` 提供了必要的患者背景信息，辅助规则引擎在特定分支（如高危人群判定）中做出决策。

### 3.3.2 核心 Schema 角色定义

系统定义了四个关键的 JSON Schema，分别承担不同的语义职责：
1. **`radiology_fact_schema`**：记录单份放射报告中提取的结构化事实。它以结节提及（Mention）为基本单位，包含 `density_category`、`has_size`、`size_mm` 和 `location_lobe` 等字段。为了保证数据的一致性，所有枚举值（如密度类别）均在 Schema 中进行了严格固定，有效避免了 "ggo"、"ground glass" 与 "ground_glass" 等同义词混用的问题。
2. **`smoking_eligibility_schema`**：记录患者级的吸烟史与高危属性。考虑到数据脱敏的现实，该 Schema 特别设计了证据质量评估字段，用于标注信息的可靠程度。
3. **`recommendation_schema`**：定义规则引擎的输出格式。除了最终的随访建议外，还必须包含 `guideline_anchor`（指南依据）和 `reasoning_path`（推理路径），以实现决策的透明化。
4. **`case_bundle_schema`**：作为系统的**唯一统一实验单位**，它将上述所有信息聚合在一起。在多源异构数据的处理中，这种设计至关重要。它避免了不同模块之间因粒度不一致（如有的按报告粒度、有的按出院小结粒度、有的按患者粒度）而导致的逻辑混乱。它是规则引擎的唯一输入源，确保了推理过程不直接依赖于原始文本，从而提高了系统的可复现性。

具体的 Schema 字段定义详见附录 X。

### 3.3.3 缺失值处理与质量标注

在真实临床数据中，信息的缺失是常态而非例外。为此，`case_bundle` 在设计上引入了两个关键机制：
- **`missing_flags` 设计**：每个 Schema 都包含一个 `missing_flags` 数组，用于记录缺失的字段。这使得系统能够区分“原始报告中未提及该信息”与“模型未能成功抽取该信息”两种不同的失效模式，为后续的误差分析提供依据。
- **`label_quality` 三级分类**：系统根据信息的完整度将病例分为 `silver`（具有高置信度结节特征与推荐线索）、`weak` 和 `unlabeled` 三个等级。只有达到 `silver` 标准的病例才会进入最终的规则推理环节，这种分级机制有效过滤了低质量数据对系统评估的干扰。

通过这种严谨的 Schema 设计，系统将复杂的非结构化医疗文本转化为了清晰、可计算的逻辑对象，为后续的符号化推理奠定了坚实的基础。

## 3.4 放射学信息抽取方法

放射学信息抽取模块是神经感知层的核心，其任务是从非结构化的放射学报告中精准提取影响随访决策的关键结节特征。本节将详细阐述该模块的输入处理流程、目标字段定义、基线方法以及基于预训练语言模型的主模型方法。

### 3.4.1 输入处理与提及级分割

本研究的数据来源为 MIMIC-IV-Note 数据库中的 `radiology.csv.gz`（包含约 232 万条记录）及 `radiology_detail.csv.gz`。为了从海量报告中高效提取结节信息，系统设计了多级预处理流程：
1. **报告过滤与初筛**：首先通过 `exam_name` 字段过滤出胸部 CT 检查记录。随后，利用结节相关关键词（如 "nodule", "mass", "opacity" 等）对报告全文进行扫描，筛选出约 19 万条含有结节描述的候选报告。
2. **段落解析（Section Parsing）**：放射学报告通常包含 FINDINGS（发现）与 IMPRESSION（印象）两个核心章节。本研究通过段落解析器提取这两个章节的文本，作为后续抽取的上下文。
3. **提及级分割（Mention-level Segmentation）**：由于一份报告中可能同时描述多个结节，系统引入了 `segment_nodule_mentions()` 函数。该函数基于结节关键词及其延续模式（Continuation Pattern），将长文本分割为独立的句子级或短语级提及（Mention），作为模型的基本处理单元。
4. **Section-aware 窗口设计**：在构建模型输入时，本研究特别采用了 Section-aware 窗口设计。实验表明，基于 Mention 为中心的局部窗口特征显著优于全文特征。这种设计能有效减少报告中非结节描述（如心脏、纵隔等其他器官的观察）对模型预测的干扰。具体的消融实验结果将在第五章详细讨论。

### 3.4.2 目标字段定义

抽取模块的目标是将每个结节提及映射到四个核心字段：
- **`density_category`**：结节密度类别，分为实性（solid）、部分实性（part_solid）、磨玻璃（ground_glass）、钙化（calcified）及不明确（unclear）五类。这是抽取任务中最具挑战性的字段。
- **`size_mm`**：结节的最大径数值，以毫米（mm）为单位。
- **`location_lobe`**：结节所在的肺叶位置，包含右肺上叶（RUL）、右肺中叶（RML）、右肺下叶（RLL）、左肺上叶（LUL）、左肺下叶（LLL）、舌叶（lingula）、双侧（bilateral）、不明确（unclear）及无位置（no_location）九类。
- **`has_size`**：二分类字段，用于判断提及中是否包含具体的尺寸信息。

### 3.4.3 基线方法与银标准生成

为了建立性能基准并为深度学习模型提供训练数据，本研究首先实现了一套基于正则表达式和临床关键词的规则基线系统（Regex Baseline）。
- **特征提取逻辑**：`extract_size()` 函数支持 mm/cm 单位换算、2D/3D 尺寸的最大值提取以及范围值（如 "2-3 mm"）的均值计算。例如，针对尺寸的正则匹配模式包括 `\d+\.?\d*\s*mm` 以及处理多维尺寸的 `\d+\.?\d*\s*x\s*\d+\.?\d*\s*mm`。`extract_density()` 与 `extract_location()` 则通过关键词匹配将文本映射到标准枚举值。
- **银标准（Silver Standard）生成**：规则基线系统在全量无标注数据上运行，生成的结构化结果被视为“银标准”标签。这种弱监督范式使得系统能够在大规模语料上进行初步的特征对齐。

此外，本研究还引入了基于传统机器学习的基线方法，包括 TF-IDF 特征提取配合逻辑回归（ml_lr）与线性支持向量机（ml_svm），参数设置为 `max_features=10000` 且 `ngram_range=(1,2)`。

### 3.4.4 基于 PubMedBERT 的主模型方法

针对规则系统在处理同义替换与复杂句式时的局限性，本研究采用了 PubMedBERT（microsoft/BiomedNLP-BiomedBERT-base-uncased-abstract-fulltext）作为主模型。
- **选择理由**：PubMedBERT 在大规模生物医学文献上进行了预训练，相比通用领域的 BERT 模型，它对医学术语（如 "ground-glass opacity", "spiculated"）具有更深层的语义理解能力，能够更好地捕捉临床文本中的细微差别。
- **训练配置**：模型以提及级文本作为输入，最大长度设置为 128。训练过程持续 5 个 epoch，采用 `batch_size=16`，学习率为 `2e-5`，并设置了 `warmup_ratio=0.1`。为了防止过拟合，引入了 Early Stopping 策略（`patience=3`）以及 `weight_decay=0.01`。为了应对类别不平衡问题，在密度与位置分类任务中启用了 `compute_class_weight("balanced")`。
- **数据规模与切分**：训练集包含 288,894 条提及，涉及 33,149 名患者。数据按患者 ID（`subject_id`）进行 70/15/15 的比例切分，以确保模型评估的公正性。

### 3.4.5 弱监督训练范式与局限性分析

本研究采用的弱监督训练范式旨在让 PubMedBERT 学习并泛化规则基线的决策边界。初步实验表明，模型在银标准测试集上的表现接近规则基线本身，但在人工标注的金标准（Gold Eval）评测中，性能出现了显著回落。
- **密度抽取的难点**：`density_category` 在金标准上的表现回落。这主要是由于真实报告中存在大量多密度混合表述（如 "solid and ground-glass"）以及隐式密度描述（如 "spiculated" 隐含实性特征），而规则基线难以覆盖这些复杂语义。
- **位置抽取的挑战**：在双侧或多叶位场景下，模型容易出错。例如，"scattered throughout both lungs" 应识别为 `bilateral`，但常被误判为 `no_location`。
- **银标准天花板现象**：初步验证表明，所有五种方法（包括 PubMedBERT 与传统机器学习基线）在金标准集上的密度预测表现出高度的一致性。这说明 PubMedBERT 等模型已完全习得并拟合了正则规则的判定逻辑。在 Regex 无法提取的样本上，PubMedBERT 仅能将极少数样本重新分类。这一发现证实了弱监督学习在处理高难度临床特征时的局限性，也凸显了神经符号架构中符号层进行逻辑兜底的必要性。模型展现了模式泛化的能力，但尚未能完全突破规则基线设定的语义边界。

## 3.5 吸烟史与高危筛查资格模块

在肺结节随访指南中，患者的吸烟史是判定其是否属于高危人群的关键因素。然而，在 MIMIC-IV 等脱敏数据集中，吸烟史信息的获取面临极大的挑战。

### 3.5.1 数据限制与脱敏现状

审计发现，MIMIC-IV 出院小结中的社会史（Social History）段落有 97.9% 的内容被强制脱敏为 `___`。具体的吸烟包年数（pack-year）数据仅在约 0.5% 的病例中可查。这种严重的信息缺失使得构建高精度的吸烟史预测模型变得极不现实。

### 3.5.2 模块定位与实现方式

基于上述限制，本模块被定位为系统的辅助属性组件，而非主贡献点。其实现遵循以下逻辑：
1. **多级提取策略**：系统优先从出院小结的 Social History 段提取吸烟状态。若该段缺失或被脱敏，则启动回退机制（Full-text Fallback）。在该策略下，系统会在全文范围内搜索含有烟草语义的句子。初步实验表明，在约 20,000 份出院小结样本中，Fallback 策略额外捕获了约 1,210 条有效信息，相比仅依赖 Social History 段落捕获的约 193 条信息，覆盖率提升了约 6 倍。为了防止歧义，系统特别排除了含有 "postpartum" 或 "TB" 等可能干扰 PPD（包/天）判定的上下文。
2. **高危资格判定**：系统基于 USPSTF 2021 标准进行判定，即：年龄在 50-80 岁之间，且满足“累计吸烟 ≥20 包年且当前吸烟或戒烟时间 ≤15 年”的患者被视为具备高危筛查资格。
3. **证据质量评估**：系统为每个推断结果标注了四级证据质量（Evidence Quality）：
   - **High**：Social History 段落清晰记录了具体的包年数或吸烟状态。
   - **Medium**：通过 Full-text Fallback 捕获到明确的吸烟描述，且无语义歧义。
   - **Low**：仅捕获到模糊的吸烟相关词汇，或存在潜在的脱敏干扰。
   - **None**：未发现任何吸烟相关证据，或关键信息完全被脱敏掩码覆盖。

需要强调的是，尽管 Fallback 策略显著提升了召回率，但受限于数据本身的脱敏质量，该模块本质上仍是一个基于规则的弱监督基线系统，其输出结果主要用于辅助下游决策，而非作为高精度的独立特征。

### 3.5.3 保守退回机制

为了确保系统的安全性，本模块引入了核心的保守退回机制。当数据不足以支撑确切的吸烟状态判定时，系统会输出 `unknown`，并在 `case_bundle` 中标注 `is_desensitized` 和 `uncertainty_level`。下游的规则引擎在接收到 `unknown` 信号时，会自动按照常规风险（非高危）逻辑进行处理。这种设计避免了模型在缺乏证据时进行盲目猜测，保证了整个神经符号链路的逻辑闭环。

## 3.6 规则推理与随访建议生成模块

在神经感知层完成特征提取后，系统进入符号决策层。本模块的核心是基于 Lung-RADS v2022 标准构建的确定性规则引擎。

### 3.6.1 规则引擎实现逻辑

规则引擎的执行流程分为以下步骤：
1. **候选结节展平**：从 `case_bundle` 中提取所有候选结节，并将其特征进行归一化处理。
2. **逐结节评估**：引擎根据结节密度进入不同的决策分支（实性、部分实性、磨玻璃、钙化）。在每个分支内，根据结节的尺寸阈值判定其 Lung-RADS 分类。
3. **修饰符规则应用**：引入动态修饰逻辑，如“稳定 ≥2 年”可导致分类降级，“显著增长”则导致分类上调。
4. **主导结节选择**：在多个结节并存的情况下，系统选择分类等级最严重且尺寸最大的结节作为主导结节，决定最终的随访建议。

### 3.6.2 随访建议映射与可追溯性

系统将 Lung-RADS 分类映射为具体的临床建议。具体的分类阈值与建议映射如表 3-1 所示。

表 3-1 Lung-RADS v2022 核心分类阈值与随访建议映射表
| 密度类型 | 尺寸条件 (最大径) | Lung-RADS 分类 | 随访建议 |
| :--- | :--- | :--- | :--- |
| 实性 (Solid, 非新发) | < 6 mm | Category 2 | 12 个月 LDCT (常规筛查) |
| 实性 (Solid, 非新发) | 6 - 7 mm | Category 3 | 6 个月 LDCT (短期随访) |
| 实性 (Solid, 非新发) | 8 - 14 mm | Category 4A | 3 个月 PET-CT 或 LDCT |
| 实性 (Solid, 非新发) | ≥ 15 mm | Category 4B | 立即组织采样 (活检) |
| 部分实性 (Part-solid) | ≥ 6 mm (无实性成分信息) | Category 4A | 3 个月 PET-CT 或 LDCT (保守处理) |
| 磨玻璃 (Ground-glass) | < 30 mm | Category 2 | 12 个月 LDCT (常规筛查) |
| 磨玻璃 (Ground-glass) | ≥ 30 mm | Category 3 | 6 个月 LDCT (短期随访) |
| 钙化或含脂肪 (Calcified/Fat) | 任何尺寸 | Category 2 | 12 个月 LDCT (良性表现) |

在处理缺失信息时，规则引擎采取了以下保守策略：
- **密度缺失处理**：当 `density_category` 缺失时，系统会默认按实性（Solid）结节进行保守处理，并在输出中记录 `uncertainty_note`，提示医生该建议基于最保守的密度假设。
- **尺寸缺失处理**：若 `size_mm` 缺失且无法通过上下文推断，系统将触发 `insufficient_data` 状态，返回“数据不足，无法生成建议”的提示，并要求人工介入。

为了增强临床信任，规则引擎在输出建议的同时，强制生成以下辅助信息：
- **`guideline_anchor`**：提供具体的指南原文依据。
- **`reasoning_path`**：记录完整的推理步骤列表，实现决策过程的 100% 可回放。
- **`triggered_rules`**：列出所有被触发的规则 ID。

这种符号化推理方式确保了建议生成过程的透明度，有效解决了黑盒模型无法解释其决策依据的问题。在一致性与可追溯性方面，该方法相比端到端生成模型具有显著优势。

## 3.7 共享尺寸解析与工程补偿策略

在肺结节特征中，尺寸数值（`size_mm`）是决定随访频率的最关键量化指标。然而，医疗报告中尺寸表述的多样性（如 "6 x 5 mm", "0.8 cm", "1.2x1.0x0.9cm"）给直接建模带来了巨大挑战。

### 3.7.1 问题的工程本质

如果要求 PubMedBERT 等预训练模型直接输出换算后的浮点数，模型往往会在单位换算、多维尺寸比较等逻辑推演上表现出不稳定性。这并非模型语义理解能力的缺失，而是序列标注任务与数值计算逻辑之间的天然错配。

### 3.7.2 共享解析器的设计

为此，本研究设计了共享尺寸解析器（Shared Size Parser）作为工程补偿策略。在该架构下：
- **模型职责**：仅负责判断 `has_size` 并提取提及尺寸的文本边界（Mention Text）。
- **解析器职责**：所有方法（Regex/ML/PubMedBERT）共享同一个 Python 解析模块。该模块负责执行 mm/cm 换算、2D/3D 取最大值以及范围值取均值等确切计算。

解析器内部维护了一套严格的处理优先级逻辑：
1. **范围值优先（Range Priority）**：若识别到范围描述（如 "2-3 mm"），解析器会取其均值（2.5 mm）作为最终尺寸。
2. **三维尺寸优先（3D Priority）**：若识别到三维描述（如 "1.2x1.0x0.9 cm"），解析器会提取其中的最大径（1.2 cm）并换算为毫米（12 mm）。
3. **二维尺寸优先（2D Priority）**：若识别到二维描述（如 "6 x 5 mm"），解析器会提取其中的最大径（6 mm）。
4. **单一数值（Single Value）**：若仅识别到单一数值（如 "0.8 cm"），则直接进行单位换算（8 mm）。

这一设计将“模式识别”与“逻辑计算”进行了有效分离。因此，本研究对 `size_mm` 的评测结果，本质上反映的是各模型在提取尺寸边界方面的稳定性，而非其独立进行数值推理的能力。这种工程补偿策略显著提升了系统在处理量化指标时的鲁棒性。它并非模型能力的飞跃，而是通过合理的工程架构设计，规避了深度学习模型在精确数值处理上的短板，确保了整个神经符号链路的稳健性。

## 3.8 本章小结

本章详细阐述了基于神经符号架构的肺结节随访建议生成方法。通过对 3.1 节至 3.7 节的论述，本章的主要贡献可以概括为以下几点：
1. **架构创新**：提出了神经-符号解耦架构，有效结合了 PubMedBERT 在语义理解上的优势与规则引擎在逻辑推理上的严密性。
2. **中间表示设计**：设计了以 `case_bundle` 为核心的结构化中间表示，确立了多源异构数据的统一实验单位与接口契约。
3. **感知层优化**：通过 Section-aware 窗口设计与弱监督训练范式，提升了放射学特征抽取的稳定性，并深入分析了“银标准天花板”现象。
4. **鲁棒性增强**：引入了针对吸烟史脱敏的保守退回机制以及针对尺寸数值的共享解析器工程补偿策略。
5. **决策透明化**：基于 Lung-RADS v2022 的规则引擎确保了随访建议生成的 100% 可追溯性与规则一致性。

在确立了系统架构与方法论后，如何科学地评估该系统在真实临床语料上的表现，将是下一章讨论的重点。第四章将详细介绍数据集的构建过程、双层评测协议的设计以及具体的实验环境配置。
