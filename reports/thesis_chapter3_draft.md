# 第三章 基于神经符号架构的肺结节随访建议生成方法

## 3.1 问题定义与总体技术路线

在真实的临床诊疗场景中，放射科医生出具的胸部影像报告与出院小结通常以非结构化的自然语言形式存在，且表述风格多样、句法结构复杂。本项目致力于解决的核心问题是：如何从这些非结构化、含有大量噪声与模糊表述的脱敏医疗文本中，稳定且可回溯地提取肺结节的关键特征，并据此自动生成符合医学指南（如 Fleischner 学会指南）的标准化随访建议。

面对这一目标，如果将其简单抽象为一个端到端（End-to-End）的普通文本生成任务，直接要求预训练语言模型阅读报告并输出随访结论，系统将面临严重的“黑盒”风险。医学决策对逻辑严密性和确定性有极高要求，而纯生成式模型容易出现“幻觉”、缺乏严格的规则一致性约束，且生成的建议无法向医生提供明确的推理溯源路径。

基于此，本项目提出并采用了一种**“神经-符号解耦”（Neuro-Symbolic Decoupling）**的总体技术路线。该架构将整个任务拆解为两个阶段：“神经感知层”（Neural Perception）负责阅读非结构化文本并完成结构化特征的信息抽取；“符号推理层”（Symbolic Reasoning）负责接收结构化特征，并严格按照指南逻辑执行确定性推理。这种设计既利用了模型在复杂文本模式识别上的优势，又通过显式的规则引擎保证了系统输出的一致性与可追溯性。

## 3.2 系统总体架构

基于上述技术路线，本系统由多个关键模块级联而成，整体工作流表现为一条清晰的数据流水线（Pipeline）。系统总体架构如图 3-1 所示。

系统的核心模块及其衔接关系如下：
1. **Radiology 信息抽取模块**：作为系统的核心感知组件，负责处理放射学报告，提取肺结节的密度、大小、位置等核心形态学特征。
2. **患者属性与高危筛查资格模块（Smoking Eligibility Module）**：作为辅助属性提取组件，负责从出院小结中推断患者的吸烟史及年龄等高危因素。受限于真实医疗数据的脱敏处理，该模块被定位为弱监督辅助模块。
3. **结构化中间表示与病例组装模块（Case Bundle Assembler）**：将前序模块在不同粒度（报告级、患者级）提取的信息，统一转化为标准化 JSON Schema，并组装为病例级运行实例（Case Bundle）。
4. **共享尺寸解析器（Shared Size Parser）**：作为工程补偿模块，介于感知层与推理层之间，负责处理所有与数值换算相关的计算逻辑。
5. **符号化推理与建议生成模块（Rule Engine / Recommendation）**：接收组装完成的 Case Bundle，应用确定的临床指南规则树，输出最终的随访建议。

## 3.3 结构化中间表示设计

为了彻底解耦“感知抽取”与“规则推理”过程，必须在两者之间建立严谨的数据契约（Data Contract）。如果缺乏统一的中间表示，不同抽取模块的字段命名、缺失值处理策略极易产生冲突，导致规则引擎难以稳定运行。

为此，本项目设计了一套层次化的 Schema 体系，包括 `radiology_fact`、`smoking_eligibility`、`recommendation_output` 以及统一的 `case_bundle` 容器。
- **模块间状态隔离**：`radiology_fact` 专注于单份放射报告的形态学事实，而 `smoking_eligibility` 专注患者的既往史。两者在 `case_bundle` 中完成患者粒度（Subject-level）的对齐聚合，使后续规则引擎可以仅依赖 `case_bundle` 进行推理，而无需再次接触原始文本。
- **缺失与不确定性表达**：由于真实数据中信息缺失严重，Schema 中显式引入了 missing flags 和不确定性标识（如 `unknown`）。这种设计使得规则引擎能够明确区分“模型未能抽取的信息”与“原报告中确实未提及的信息”，从而制定不同的降级或保守兜底策略。
- **可复现与可解释价值**：统一的结构化中间表示为后续构建评测基准（Benchmark）提供了标准单位，也使得预测错误能够被精确隔离和分析（即可以明确是抽取模块出错，还是规则引擎路径出错）。

## 3.4 Radiology 信息抽取方法

Radiology 信息抽取模块是系统神经感知层的核心。该模块的输入为经过段落解析（如提取 FINDINGS 与 IMPRESSION 章节）后的放射学报告，采用实体提及级别（Mention-level）的抽取策略。

其目标字段主要包含影响结节危险度分级的四个核心维度：结节密度（`density_category`，分为实性、部分实性、磨玻璃等）、尺寸存在性（`has_size`）、具体尺寸数值（`size_mm`）以及所在叶位（`location_lobe`）。

在方法演进上，本模块采取了由规则到模型的渐进式方案：
1. **规则基线与银标准生成**：首先利用基于大量临床知识构建的复杂正则表达式（Regex Baseline）对全量无标注数据进行特征提取，并将提取结果视为“银标准（Silver Standard）”弱监督信号。
2. **预测模型学习与泛化**：随后，引入轻量级传统机器学习分类器（如基于 TF-IDF 的 LR/SVM）以及预训练语言模型（如 PubMedBERT），在银标准数据上进行训练。其目的在于让模型学习并泛化规则的决策边界，以处理规则难以覆盖的同义替换与模式变体。

在任务特性方面，实验表明不同特征的抽取难度差异显著：
- **密度的复杂性**：`density_category` 是本模块中最具挑战性的字段。真实报告中常出现多密度混合描述（如“部分实性结节内含磨玻璃成分”），要求模型具备较强的长文本模式识别能力。
- **位置的模糊性**：`location_lobe` 字段的难点集中于双侧（Bilateral）或多叶位（Multi-lobe）同时发生结节的场景，报告通常使用泛指代词而非明确的解剖学定位，对提及级抽取造成了显著干扰。

## 3.5 患者属性与高危筛查资格模块

在 Fleischner 指南中，患者是否具备高危因素（如年龄与长期吸烟史）是决定随访频率的重要分支条件。然而，在处理 MIMIC-IV-Note 等真实脱敏医疗数据集时，出院小结中的社会史（Social History）及具体的吸烟包年数（Pack-year）遭遇了大面积的强制掩码脱敏。

面对这一客观数据限制，本模块未采取激进的强预测方案，而是被设计为一个基于规则辅助与弱监督匹配的模块，并引入了核心的**保守退回机制（Fallback Strategy）**。
当从文本中获取到明确的吸烟史提示时，系统提取对应属性；但当遭遇脱敏掩码或信息极度缺失时，系统会合法地输出 `unknown` 或不确定标识。这种设计保证了系统在面临高危证据不足时，会自动在下游规则引擎中回退至常规风险处理逻辑。本模块的主要定位是为系统提供必要的辅助属性拼图，而非承担高精度独立建模的主任务。

## 3.6 共享尺寸解析与工程补偿策略

在特征抽取中，结节的最大径（`size_mm`）是判断危险等级的首要量化指标。然而，报告中的尺寸表述格式极其繁杂（如 “6 x 5 mm”, “0.8 cm”, “1.2x1.0x0.9cm”）。如果直接要求序列标注模型或预训练语言模型输出换算后的毫米级浮点数，极易暴露模型在数学计算与逻辑推演上的天然弱点，导致严重的数值不稳定。

为解决这一问题，本项目设计了一套极具特色的**“共享尺寸解析模块”（Shared Size Parser）**，作为一种工程补偿策略。
在该机制下，感知层的预测模型仅负责在长文本中圈定并提取“提及该尺寸的局部原文边界”（Mention Text）；而后续具体的单位统一、多维尺寸的最大径筛选以及毫米换算，均统一交由基于 Python 逻辑的独立解析器处理。

这一工程决策将“模型擅长的模式匹配”与“代码擅长的确切计算”进行了完美结合。这一口径也决定了本研究对 `size_mm` 字段的评测，本质上反映的是各模块在共享尺寸解析器支撑下的“文本边界提取稳定性”，而非不同模型在数值计算维度的独立建模能力。

## 3.7 符号化指南推理模块

当 `case_bundle` 完成所有结构化信息的聚合后，数据将流入符号化推理模块（Recommendation Engine）。

在这一阶段，系统完全摒弃了生成式模型。原因在于，医疗随访建议必须遵循严格的指南树逻辑，不允许任何不可控的偏离。模块将 Fleischner 肺结节随访指南转化为显式的条件判断规则库。引擎读取 `case_bundle` 中的密度、最严尺寸、患者年龄与吸烟史，确定性地落入对应的分支节点。

最终生成的输出不仅包含类似“3-6个月复查CT”的具体建议，还被强制要求附带触发建议的“指南锚点”（Guideline Anchor）与“推理路径”（Reasoning Path）。这种符号化约束确保了系统具备 **100% 的规则一致性**与清晰的**可追溯性**，在很大程度上缓解了黑盒 AI 系统在临床应用中的信任危机。

## 3.8 本章小结

本章详细介绍了本项目所采用的神经-符号解耦架构，以及从原始医疗文本到最终随访建议的完整处理链路。通过对 Radiology 信息抽取模块的设计分析，以及共享尺寸解析器和保守退回机制的引入，本系统在工程实现上有效规避了纯预训练语言模型的固有缺陷，通过 Schema 契约和符号化推理确保了输出的临床逻辑一致性。

在确立了本架构与方法后，如何有效地构建训练数据、并在缺乏大规模金标准的前提下公正客观地评测该系统的各模块能力，将是下一章的讨论重点。第四章将详细阐述基于真实脱敏语料的数据集构建方法与双层评测协议设计。
